# Microsoft Sentinel Interview Questions

## Basic Questions

### What is Microsoft Sentinel?

Microsoft Sentinel is a cloud-native SIEM and SOAR platform used for threat detection, investigation, and response.

---

### What does SIEM stand for?

Security Information and Event Management.

---

### What does SOAR stand for?

Security Orchestration, Automation and Response.

---

### Difference between SIEM and SOAR?

SIEM focuses on log collection, monitoring, and threat detection.

SOAR focuses on automation and response actions.

---

### What is a Log Analytics Workspace?

The central repository used to collect, store, and query logs in Microsoft Sentinel.

---

### What are Data Connectors?

Data Connectors ingest logs from data sources into Microsoft Sentinel.

---

### What is KQL?

Kusto Query Language used for searching and analyzing logs.

---

### What are Analytics Rules?

Detection rules that generate alerts based on log activity.

---

### What is an Alert?

A single security detection generated by an analytics rule.

---

### What is an Incident?

A collection of related alerts grouped for investigation.

---

### What is Investigation Graph?

A visual tool used to analyze relationships between entities involved in an incident.

---

### What are Workbooks?

Interactive dashboards used to visualize security data.

---

### What are Automation Rules?

Rules that automatically perform actions when incidents or alerts are generated.

---

## Intermediate Questions

### How does Microsoft Sentinel collect logs?

Through Data Connectors.

---

### Where are logs stored?

In Log Analytics Workspace.

---

### How are incidents generated?

Analytics Rules generate alerts which can be grouped into incidents.

---

### What are common Sentinel entities?

* User
* Host
* IP Address
* URL
* Process

---

### What is Fusion Detection?

Microsoft-built correlation engine that combines multiple alerts into higher confidence incidents.

---

### What is UEBA?

User and Entity Behavior Analytics used to identify abnormal behavior patterns.

---

## Scenario Questions

### How would you investigate multiple failed logins?

1. Review incident.
2. Check source IP.
3. Check target account.
4. Review login history.
5. Determine if brute force activity exists.
6. Escalate if required.

---

### How would you investigate suspicious PowerShell activity?

1. Review process details.
2. Check command line.
3. Identify parent process.
4. Review related alerts.
5. Determine malicious intent.

---
