# Microsoft Sentinel Overview

## What is Microsoft Sentinel?
Microsoft Sentinel is a cloud SIEM platform used for security monitoring, incident investigation, threat hunting, and automation.

## Why SOC Analysts Learn Sentinel
SOC analysts use Sentinel to:
- Collect logs from cloud and endpoint sources
- Write KQL queries
- Investigate incidents
- Review alerts
- Create analytics rules
- Perform threat hunting
- Map detections to MITRE ATT&CK

## Key Components

| Component | Purpose |
|---|---|
| Log Analytics Workspace | Stores collected log data |
| Data Connectors | Connect Microsoft and third-party data sources |
| Analytics Rules | Detection rules that generate alerts/incidents |
| Incidents | Grouped alerts requiring investigation |
| Hunting | Manual query-based threat hunting |
| Workbooks | Dashboards and visual reports |
| Automation Rules | Automated response actions |
| Playbooks | Logic App based response workflows |

## Interview Talking Point
Microsoft Sentinel is a cloud SIEM used by SOC teams to collect logs, create analytics rules, investigate incidents, run KQL queries, and perform threat hunting across cloud and endpoint environments.
